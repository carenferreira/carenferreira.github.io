Veículo Arduino Controlado por API

Neste projeto, desenvolvemos um robô Arduino controlado por uma API HTTP e que utilizará programação em blocos visuais. Abaixo, explicamos cada ponto do projeto.

Previous Next

Principais Recursos

O carrinho que desenvolvemos no trabalho foi programado através com Arduino e, além dos movimentos básicos (andar para frente, para trás e para os lados, por exemplo), ele também possui com um sensor de distância frontal para detectar obstáculos em seu caminho. O carrinho conta com um farol para iluminar o que estiver à sua frente, emite um sinal sonoro de ré quando o comando para tal é dado e, por fim, possui um LED RGB que indica seu estado de conexão à API: verde caso esteja conectado e vermelho caso não possua conexão.

Apresentação do Projeto

Neste vídeo de apresentação, nós explicamos a estrutura do carrinho e seu funcionamento, seus recursos, e elaboramos mais sobre a API HTTP desenvolvida para a comunicação com o carrinho. Também é explicado o uso da biblioteca Blocky, utilizada para a programação em blocos visuais, e como nós integramos estes recursos.

Documentação da API

A documentação da API explica de maneira mais profunda como nós desenvolvemos a API o funcionamento desta em relação ao carrinho e a comunicação com este. A documentação conta com exemplos e detalhes de como decidimos integrar cada função e também explica como utilizar o carrinho, além de explicitar os recursos utilizados durante o desenvolvimento do projeto.

Interface de Programação Blockly

Nós utilizamos a biblioteca Blocky para implementar uma interface de programação em blocos visuais. Ela fornece uma maneira fácil e intuitiva de controlar o veículo, sem a necessidade de conhecimento avançado sobre programação tradicional. Através do Blocky, nós conseguimos criar lógicas complexas sem tornar o uso do carrinho excessivamente complicado, mantendo a interface visual simples e eficiente.

Desafios do Mundo Real

O carrinho será testado em uma situação simulando o "mundo real", movendo-se por pistas com obstáculos criadas pelo professor a fim de descobrir se o carrinho conseguiria funcionar fora de situações perfeitamente controladas. O "teste de fogo" será descobrir se os algoritmos de blocos e a construção do carrinho em si serão suficientes para percorrer o percurso corretamente, demonstrando a viabilidade do projeto na prática.
