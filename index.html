Veículo Arduino controlado via API
Objetivo do trabalho

O trabalho proposto na disciplina de Webservices (4°período), tinha como desafio o desenvolvimento de: um veículo robô Arduino, uma API HTTP e uma interface de programação em blocos visuais. Sendo a API HTTP responsável pela comunicação com o veículo, e a interface de programação em blocos com a API, o que permite que o usuário controle o veículo.

Requisitos do veículo

* Ser capaz de andar para frente, para trás e virar para os dois lados;

* Deverá também possuir como recursos mínimos: sensor de distância na parte frontal, luz e sinal sonoro de ré, e um led RGB para indicar seus status (conectado/verde ou desconectado/vermelho à API).

Requisitos da API

possibilite enviar informações para o robô, como andar para frente, virar para a esquerda, checar se há algum objeto à frente, etc. A comunicação remota do robô poderá utilizar o módulo de Bluetooth ou o de Wi-Fi (ambos disponíveis no Laboratório de Robótica).

Desenvolvimento

Para o desenvolvimento do trabalho, usamos a documentação da API Blockly. Para mais informações sobre o desenvolvimento e como executar o projeto, basta seguir as instruções do README.

Para controlar o carrinho, basta o usuário montar uma sequência de movimentos usando os blocos predefinidos na seção de movimentos e, logo após, pressionar o botão para executar os movimentos.
